NOW THE IMPORTANT PART
WHICH ELEMENTS HAVE TEXT AND WHICH DO NOT?
Element	Has direct text?	Has nested text?	Meaning
Avatar	no	no	image-only
Username	yes	yes	clickable text
Verified badge	no	no	icon-only
Follow button	no	svg title only	icon-only
Fediverse button	no	svg title only	icon-only
Timestamp	no	yes	text inside <time>
Post text	yes	yes	main content
Media	no	no	image
Like	no	count nested	icon + number
Reply	no	no	icon-only
Repost	no	no	icon-only
Share	no	count nested	icon + number
Markup	no	no	icon-only
More (3 dots)	no	no	icon-only

So yes — MOST Threads buttons have zero text.
That’s the platform design.

WHAT YOUR EXTENSION MUST USE INSTEAD OF TEXT

When there is no text, extension should rely on:

role="button"

SVG title=""

aria-label

cluster position

icon shape

size and geometry patterns

interaction areas around the SVG

That’s enough for 100% BBX mapping.
